# 不休眠猫头鹰设置与中英文切换设计

## 目标

增加一个独立的 macOS 原生设置窗口，让用户选择简体中文或英文，并分别控制电脑热状态与高占用应用信息是否显示。所有设置修改后立即生效，并在应用重启后保留。

## 用户入口

- 右键菜单在“打开不休眠猫头鹰…”下方增加“设置…”；英文状态显示为 “Settings…”。
- 设置窗口使用独立 `NSWindow`，标题随语言显示为“不休眠猫头鹰设置”或 “No Sleep Owl Settings”。
- 重复点击设置入口时复用并置前已有窗口。

## 设置项

### 语言

- 支持“简体中文”和 “English”。
- 默认值为简体中文，保留当前用户体验。
- 修改后立即刷新控制窗口、右键菜单、设置窗口、通知文案、错误提示和辅助程序状态文案。
- 应用名称与系统安装包名称继续使用“不休眠猫头鹰”，避免切换语言后产生两个应用身份。

### 显示电脑热状态

- 默认开启。
- 开启时，控制窗口显示热状态标题与说明，右键菜单显示“电脑状态”摘要。
- 关闭时，隐藏热状态标题、说明以及右键菜单的热状态摘要。
- 热状态采样、严重/危急通知和自动安全保护继续运行。

### 显示高占用应用

- 默认开启。
- 开启时，控制窗口显示前 7 名 CPU 应用、占用比例和安全退出按钮；右键菜单的电脑状态摘要附带占用最高的应用。
- 关闭时，隐藏应用排行、CPU 数值和退出按钮。
- CPU 采样在隐藏时暂停，降低后台资源消耗；重新开启后立即启动首轮采样。

### 组合显示规则

- 两项都开启：显示完整电脑状态卡片。
- 仅开启热状态：卡片只显示热状态标题和说明。
- 仅开启高占用应用：卡片显示“应用占用”标题和前 7 名列表，不显示热状态级别。
- 两项都关闭：控制窗口隐藏整个电脑状态卡片，右键菜单不显示电脑状态摘要。

## 架构

### `AppPreferences`

集中管理以下持久化值：

- `language`: `zhHans` 或 `en`
- `showsThermalStatus`: `Bool`
- `showsHighUsageApps`: `Bool`

使用 `UserDefaults` 保存。对外暴露当前快照与 `onChange` 回调，确保一个设置变化只触发一次全局刷新。

### `AppStrings`

提供强类型文案键，根据当前语言返回中文或英文。界面控制器和状态模型不再直接写入用户可见文案。动态内容通过带参数的方法生成，例如守夜时长、CPU 摘要和错误原因。

### `SettingsWindowController`

负责设置窗口布局与交互：

- 语言使用弹出式选择器。
- 两个显示选项使用复选框。
- 修改设置后写入 `AppPreferences`。
- 监听全局刷新，使窗口自身文案立即切换语言。

### 现有组件调整

- `AppDelegate` 创建并持有唯一的 `AppPreferences` 与 `SettingsWindowController`。
- `ControlWindowController` 根据偏好决定状态卡片的显示模式。
- `StatusItemController` 增加设置入口，并根据偏好组织摘要。
- `ThermalAppMonitor` 始终采样热状态，仅在需要展示高占用应用时采样进程 CPU。
- 安全策略继续直接读取系统热状态，不依赖界面显示开关。

## 数据流

1. 用户在设置窗口修改选项。
2. `AppPreferences` 写入 `UserDefaults` 并发出一次变化通知。
3. `AppDelegate` 统一刷新控制窗口、设置窗口、菜单栏提示和监控策略。
4. 下次打开右键菜单时使用最新语言与显示偏好即时构建内容。

## 错误与兼容处理

- 读取到未知语言值时回退到简体中文。
- 旧版本没有设置值时使用三个默认值：简体中文、显示热状态、显示高占用应用。
- 设置窗口关闭不会退出应用，也不会改变防休眠状态。
- 隐藏监控信息不会关闭过热保护。

## 测试与验收

- 偏好默认值、持久化与未知语言回退测试。
- 中英文核心文案和动态格式测试。
- 四种显示组合的页面与菜单策略测试。
- 高占用应用隐藏时暂停 CPU 采样的策略测试。
- 设置变化触发一次刷新测试。
- 完整现有测试套件回归。
- 构建 arm64 应用并替换 `/Applications/不休眠猫头鹰.app`。
- 实际验证设置入口、窗口复用、中英文即时切换、两个显示开关以及重启后保留。

## 本次范围外

- 跟随系统语言。
- 更多语言。
- 用户自定义温度或 CPU 阈值。
- 关闭严重过热通知或自动安全保护。
