# 不休眠猫头鹰：热状态与高占用应用监控设计

## 目标

在不增加管理员权限、不读取具体传感器温度的前提下，让用户看到 macOS 系统热状态和 CPU 占用最高的应用，并在过热时获得清晰、克制、可操作的提醒。

## 用户体验

- 控制页面新增“电脑状态”区域，显示“正常、偏热、严重、危急”。
- 常驻显示 CPU 占用最高的三个普通应用，包括应用名称、图标和 CPU 百分比。
- 菜单栏右键菜单显示一行摘要，例如“电脑状态：偏热 · Chrome 86%”。
- 偏热时使用橙色状态，提示用户检查高占用应用。
- 严重时使用红色状态并发送一次 macOS 通知；通知冷却时间为十分钟。
- 危急时继续执行现有保护：结束猫头鹰模式并恢复系统睡眠。
- 普通应用提供“退出”按钮。点击后先通过系统正常终止应用，不执行强制结束。
- Finder、WindowServer、系统设置和安全组件只显示，不提供退出按钮。

## 监控策略

- 每十秒采样一次，与现有安全监控周期一致。
- 使用 `ProcessInfo.thermalState` 获取系统热状态，不读取具体摄氏温度。
- 从当前运行的 GUI 应用中筛选用户应用，再通过进程采样计算 CPU 占用。
- 排除不休眠猫头鹰自身。
- 单个应用连续两次超过 80% CPU 才标记为“持续高占用”，避免瞬时尖峰误报。
- 监控只在本机内存中运行，不保存历史，不上传数据。
- 热状态监控始终运行；自动结束守夜只在猫头鹰模式生效。

## 架构

### `ThermalSnapshotProvider`

读取 `ProcessInfo.thermalState`，转换为核心层可测试的 `OwlThermalState`。

### `ApplicationUsageSampler`

维护每个应用进程的前一次 CPU 时间和采样时间，输出按 CPU 百分比排序的前三个应用。应用元数据来自 `NSRunningApplication`，进程 CPU 时间来自系统进程接口。采样失败时跳过该应用，不中断整体监控。

### `ThermalAppMonitor`

每十秒组合热状态和应用排行，维护连续高占用次数、通知冷却时间和最新快照。通过回调刷新控制窗口与菜单栏摘要。

### 控制窗口

新增独立状态卡片，负责展示热状态、前三个应用和退出操作。退出按钮调用 `NSRunningApplication.terminate()`；系统应用或不可终止应用不显示按钮。

### 菜单栏

菜单创建时读取最新快照并插入禁用的状态摘要，不增加额外轮询。

## 错误与安全边界

- 无法读取某个进程时忽略该进程。
- 无法完成一轮采样时保留上一份有效快照，并显示“正在获取应用占用”。
- 用户点击退出后不自动升级到强制退出。
- 系统热状态优先于应用 CPU 排行；危急状态即使没有进程数据也会触发现有安全恢复。
- 通知权限被拒绝时，页面内提醒继续工作。

## 测试

- 热状态到中文标题、颜色语义和提醒级别的映射。
- CPU 排序和前三名截断。
- 连续两次超过 80% 才进入持续高占用状态。
- 系统应用退出权限规则。
- 严重通知十分钟冷却。
- 危急状态仍触发现有退出守夜策略。
- 手动验证页面状态卡、菜单摘要、通知和正常退出应用。

## 非目标

- 不显示具体摄氏温度。
- 不保存温度或 CPU 历史曲线。
- 不自动结束任何用户应用。
- 不强制结束无响应进程。
- 不新增管理员权限或网络请求。
