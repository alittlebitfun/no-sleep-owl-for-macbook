# 不休眠猫头鹰 v2：免重复密码与安全保护设计

## 目标

v2 将当前每次开启🦉都需要管理员密码的临时守护脚本，升级为 macOS 管理的特权辅助程序。用户首次批准后，日常切换无需重复输入密码，同时获得电池、电源与热状态保护。

## 用户体验

- 首次使用 v2 时，应用引导用户批准“合盖守夜辅助程序”。
- 辅助程序批准完成后，左键切换🦉无需密码。
- 控制页展示辅助程序状态：未安装、等待批准、可用、异常。
- 默认电源策略为“仅接通电源时允许合盖守夜”。
- 用户可以选择“使用电池时也允许”。
- 电池供电且电量低于 20%时发送本地提醒。
- 电池供电且电量低于 10%时自动切回🐦。
- 系统热状态为 `serious` 时提醒，为 `critical` 时自动切回🐦。
- 自动退出🦉时显示明确原因。

## 特权辅助程序

应用使用 `SMAppService.daemon(plistName:)` 注册打包在主应用内的 LaunchDaemon。辅助程序运行于 root 上下文，通过受限 XPC 接口提供三项能力：

1. 开启合盖禁睡。
2. 关闭合盖禁睡。
3. 查询当前状态。

辅助程序不接受任意 shell 命令、路径或参数。它只调用固定的 `/usr/bin/pmset -a disablesleep 1|0` 操作。

主应用根据 `SMAppService.Status` 展示授权状态。若系统要求人工批准，应用打开 macOS 登录项设置页面并等待状态变为可用。

## 安全状态机

- 辅助程序开启合盖禁睡前记录原始 `SleepDisabled` 值。
- 主应用处于🦉时每 5 秒发送心跳。
- 辅助程序 15 秒没有收到心跳时恢复原始值。
- XPC 连接中断、主应用退出、辅助程序收到终止信号时恢复原始值。
- 用户切回🐦时立即恢复原始值。
- 开机时辅助程序默认不启用禁睡。
- 主应用启动时发现残留禁睡状态，会请求辅助程序恢复并显示修复结果。

## 电源与热保护

主应用使用 IOKit 电源源信息读取是否接通电源、电池百分比和充电状态；使用 `ProcessInfo.thermalState` 读取 Apple 官方热状态。

策略：

- `acOnly`：拔掉电源时自动切回🐦。
- `allowBattery`：电池供电时允许继续，低电量规则仍生效。
- 20%提醒每次🦉会话只发送一次。
- 10%自动退出无需再次确认。
- `serious` 提醒每次热状态事件只发送一次。
- `critical` 自动退出无需再次确认。

所有自动退出原因均写入应用内状态，不发送网络数据。

## 组件

- `NoSleepOwlHelper`：root LaunchDaemon，可执行固定电源操作并维护心跳超时。
- `NoSleepOwlXPCProtocol`：主应用与辅助程序共享的最小接口。
- `PrivilegedSleepController`：主应用侧的注册、连接、状态查询和模式切换封装。
- `SafetyMonitor`：读取电源和热状态，执行提醒与自动退出决策。
- `PowerPolicyStore`：持久化 `acOnly` 或 `allowBattery`。
- `ControlWindowController`：展示授权状态、电源策略和保护状态。

## 兼容与回退

- 目标系统为 macOS 15，架构为 arm64。
- 辅助程序不可用时，应用保持🐦，不回退到每次执行管理员 shell 的旧方案。
- v2 首次成功安装并验证后，删除旧版临时 marker；若检测到旧版 root 守护脚本，先让它恢复再退出。
- 应用和辅助程序均使用同一构建签名链；本机交付前验证注册、批准、XPC、心跳超时和恢复流程。

## 测试与验收

自动化测试覆盖：

- XPC 接口只暴露固定能力。
- 心跳 15 秒超时恢复原始休眠值。
- XPC 断开与辅助程序终止触发恢复。
- `acOnly` 拔电自动退出。
- 20%只提醒一次，10%自动退出。
- `serious` 只提醒一次，`critical` 自动退出。
- 辅助程序不可用或未批准时保持🐦。
- 启动残留状态修复。

实机验收：

1. 首次批准辅助程序。
2. 连续切换三次🦉/🐦，不再要求密码。
3. 🦉状态下确认 `SleepDisabled=1`，🐦状态下确认恢复原值。
4. 强制结束主应用，15 秒内确认恢复。
5. 模拟电源、电量与热状态决策并验证自动退出原因。
6. 验证 arm64、签名、Bundle 和 LaunchDaemon 注册状态。

## 发布注意

特权辅助程序是安全敏感组件。若本机 ad-hoc 签名无法满足 `SMAppService` 的注册要求，开发版将停在“辅助程序不可用”并报告真实原因，不会采用降低系统安全性的替代安装方式。正式分发需要稳定的 Developer ID 签名与公证。
