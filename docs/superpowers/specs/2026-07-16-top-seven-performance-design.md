# 前七名应用与监控性能优化设计

## 目标

将高占用应用排行从三名扩展为七名，同时降低控制窗口关闭时的后台采样开销。

## 交互

- 控制页面显示 CPU 占用最高的七个普通应用。
- 状态卡保持固定高度，应用列表在卡片内部滚动。
- 每行继续显示图标、应用名称、CPU 百分比和安全的“退出”按钮。
- 菜单栏右键摘要只显示最高占用应用。

## 性能策略

- 控制窗口可见或热状态达到“偏热、严重、危急”时，每十秒采样。
- 控制窗口关闭且热状态正常时，每二十秒采样。
- 缓存 PID 对应的可执行路径；PID 消失时清理缓存。
- 每轮仍以累计 CPU 时间差计算应用及其子进程占用。
- 排行只保留前七名，不保存历史。

## 测试

- 排行按 CPU 降序保留七名。
- 窗口可见或热状态升高时选择十秒间隔。
- 窗口关闭且热状态正常时选择二十秒间隔。
- PID 路径缓存只复用仍然存活的进程。
- 正式包页面能看到七行或通过滚动查看七行。
