# 「从使用 Agent 到组织的 Agent 化」周会视频设计

## 目标

制作一支约 60 秒、16:9 横屏的公司内部周会展示视频。重点呈现本周高强度推进过程与工作量，并将分散的研究、评测、数据、训练和交付活动收束为一条组织能力升级主线。

## 核心表达

主标题：从使用 Agent，到组织的 Agent 化。

收束句：把个人能力，变成可复制的组织能力。

叙事从“个人调用 Agent 完成研究任务”出发，逐步扩展为可拆解、可并行、可恢复、可评测、可交付的 Agent 工作系统。

## 本周任务类型

本周任务需明确区分为五类。前半段分别呈现，后半段汇入统一的 Agent 组织系统。

1. **研究与决策**：视频模型横评、Techpack 需求分析、产品策略研究
2. **批量智能作业**：大批量 VLM 评测、持续推理、错误回收与质量分桶
3. **数据与模型工程**：词典数据校验、标签体系、京东多标签训练、ControlNet 数据准备
4. **产品与组织设计**：模型训练服务、用户旅程、Agent 工作流设计
5. **交付与基础设施**：模型包交付、采购与算力材料、存储清理、运行保障

五类任务分别使用不同的视觉动词：研究采用检索与对比，批量作业采用队列与计数，模型工程采用管线与训练曲线，组织设计采用节点编排，交付与基础设施采用封装、校验和状态灯。

## 受众与观看场景

- 受众：公司内部周会参会者
- 画幅：1920 × 1080，16:9
- 时长：目标 60 秒，允许误差不超过 2 秒
- 播放：会议屏幕播放，需兼顾投影环境中的文字可读性
- 声音：冷静专业女声中文旁白，电子节拍音乐低音量铺底

## 视觉系统

- 基调：深黑、冷静、技术纪录片感
- 主文字：冷白色
- 主强调色：电蓝，用于数据流、节点和完成态
- 次强调色：荧光橙，仅用于告警、失败和返工节点
- 字体：优先使用本机可商用的中文无衬线字体；缺失时使用系统苹方
- 运动：快速窗口堆叠、数据扫描、节点连接、数字递增、短促闪白转场
- 节奏：快—快—稳—快—扩张—停顿收束
- 限制：单屏核心信息不超过两层；关键数字每次最多展示三项；不堆叠长段代码

## 60 秒分镜与旁白

### 0–6 秒：起点

画面从黑场和微弱脉冲开始，一个任务窗口出现，随后研究资料、命令和结果卡片快速叠入。

旁白：

> 最初，我们用 Agent 查资料、跑任务、整理结果。

屏幕大字：`USE AGENT`

### 6–18 秒：任务密度上升

五种任务卡从中心窗口分裂：研究与决策、批量智能作业、数据与模型工程、产品与组织设计、交付与基础设施。随后镜头进入“批量智能作业”，评测记录高速滚动，结果被自动归入成功、部分成功和失败。

旁白：

> 这周，Agent 进入五类任务：研究决策、批量作业、模型工程、组织设计和稳定交付。三千四百六十九条样本进入评测，形成可量化、可复跑的流水线。

可信数据：

- 3,469 条评测输入
- 3,464 条获得结果
- 99.86% 有效 JSON
- 2,886 success / 469 partial / 109 fail

### 18–32 秒：工作流组织化

镜头从“批量智能作业”切换到“数据与模型工程”。散乱窗口被吸入统一管线，依次通过数据校验、标签结构、数据切分和训练准备节点。

旁白：

> 数据校验、标签统一、训练集构建，每一步都有状态、检查点和明确产物。

可信数据：

- 1,841 张词典图片完成校验，缺失或失败为 0
- 56 标签统一结构
- 43,162 条京东训练记录
- 5,395 条验证记录与 5,395 条测试记录

### 32–45 秒：高强度推进

五条颜色和形态保持统一的任务轨并列出现。训练、复跑、错误回收、质量分桶、产品流程设计、模型封装和 ControlNet 数据处理在多轨时间线上同步推进。失败节点以橙色回流，完成节点保持电蓝。

旁白：

> 研究形成决策，批量作业持续运行，模型工程保留检查点，产品流程完成编排，交付结果独立验收。

补充画面信息：

- 300 组精细线稿 ControlNet 数据
- 184 条组合任务二轮评测
- 88.04% success，平均分 8.67

### 45–55 秒：组织能力出现

画面拉远，五类任务卡各自聚合为一个 Agent 节点，再连接为统一闭环。任务沿节点流转，状态持续反馈。

旁白：

> 五类任务进入同一套协作、状态和验收机制，Agent 开始成为组织工作方式。

节点标签：`RESEARCH / BATCH OPS / MODEL ENGINEERING / ORGANIZATION / DELIVERY`

### 55–60 秒：收束

音乐短暂抽空，所有节点汇聚成主标题。标题停留至少 2.5 秒。

旁白：

> 从使用 Agent，到组织的 Agent 化。把个人能力，变成可复制的组织能力。

## 音频设计

- 女声语速保持克制，约每分钟 220–250 个汉字
- 旁白总长度控制在约 190–210 个汉字，并为关键数字和结尾留出停顿
- 配乐采用渐进式电子节拍：前 6 秒低频启动，18 秒后增加律动，45 秒进入峰值，55 秒抽空后收束
- 旁白响度始终高于音乐；转场音效仅作为节奏标记
- 若本地 TTS 音色机械感明显，优先保证咬字、节奏和信息可信度

## 素材与实现边界

- 优先使用现有评测页面、图片数据、训练目录、报告页面和交付物作为视觉素材
- 可将真实日志和数据加工为可读的动态图形，不虚构运行结果
- 敏感路径、账号、内部地址和个人信息不得进入画面
- 任何未完成训练结果不得包装成已完成成果
- 最终同时输出带旁白成片和可替换旁白的工程源文件

## 验收标准

- 时长 58–62 秒
- 1920 × 1080，H.264 MP4
- 所有关键数据与当前本地结果文件一致
- 关键字幕在会议投影环境中清晰可读
- 旁白完整、无削波，音乐不遮挡人声
- 开头 6 秒建立任务强度，45 秒前完成事实铺陈，最后 5 秒准确落到主旨
- 抽查 0、6、18、32、45、55、60 秒附近画面，无文字重叠、黑帧或素材缺失
