# 不休眠猫头鹰显示位置设置设计

## 目标

在现有设置窗口中增加“显示状态栏图标”和“显示 Dock 图标”两个开关，支持运行时即时切换，并在应用重启后保留选择。

## 用户体验

- 设置窗口新增两个复选项：
  - 显示状态栏图标
  - 显示 Dock 图标
- 修改后立即生效，不要求重启。
- 状态栏图标关闭时，状态栏项目被移除；重新开启时恢复原有小鸟/猫头鹰图标与右键菜单。
- Dock 图标关闭时，应用使用菜单栏应用表现；Dock 图标开启时，应用使用普通 App 表现。
- 两个选项同时关闭时，应用继续运行，设置窗口仍可通过启动参数或现有窗口路径打开，应用不会自动退出。
- 默认值：状态栏显示开启，Dock 显示关闭，保持当前菜单栏工具体验。

## 架构

- `AppPreferences` 增加 `showsStatusBarIcon` 与 `showsDockIcon` 两个持久化字段。
- `SettingsWindowController` 增加两个复选框并绑定偏好修改方法。
- 新增 `DisplayLocationController`，集中处理状态栏项目生命周期和 `NSApplication` 激活策略。
- `AppDelegate` 监听偏好变化，将显示位置快照交给控制器；控制器保证状态栏项目只存在一个实例。
- Dock 显示通过 `NSApplication.setActivationPolicy(_:)` 切换 `.accessory` 与 `.regular`，状态栏显示通过创建/释放 `StatusItemController` 实例切换。
- 关闭状态栏图标时，不改变菜单内容和核心防休眠逻辑；重新显示时复用当前模式、语言和监控状态。

## 错误与边界

- 偏好缺失或类型不正确时，使用默认值：状态栏开启、Dock 关闭。
- 两个显示选项关闭时不触发退出，避免用户失去后台进程。
- 状态栏项目重复创建必须被阻止，避免出现多个图标或旧回调引用。
- Dock 激活策略切换失败时，保留状态栏与核心功能，并在设置页继续显示用户选择。

## 测试标准

- 偏好默认值、持久化和未知值回退有单元测试。
- 四种状态栏/Dock 组合均有策略测试。
- 设置复选框能即时修改偏好并触发显示位置控制器。
- 状态栏项目创建、隐藏、恢复不产生重复实例。
- Dock 激活策略在 `.accessory` 与 `.regular` 间切换。
- 现有防休眠、监控、语言、右键菜单和重复打开窗口测试继续通过。
- Apple 芯片 release 构建、签名和本地安装验证通过。
